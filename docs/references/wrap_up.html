Loops and If-statements in R

Why loops and if-statements exist?

Most R code works on vectors. That is powerful, but sometimes you want to:

  • do something step by step
  • repeat the same operation many times
  • make decisions depending on a condition

That is exactly what loops and if-statements are for.

Think of them as:

  • loops: repeat
  • if-statements: decide

For-loops: doing something repeatedly

A for-loop runs the same code for a sequence of values.

Basic structure

# this code wont work. It just contain the structure

for (variable in sequence) {
  code that is repeated
}

Read this literally as:

For each value in this sequence, do the following.

Example 1: Printing numbers

for (i in 1:5) {
  print(i)
}

What happens:

  • 1:5 creates the sequence 1, 2, 3, 4, 5
  • i takes one value at a time
  • print(i) is executed once per value

Example 2: Looping over a vector

trees <- c("oak", "beech", "pine")

for (tree in trees) {
  print(tree)
}

Here:

  • tree is just a name
  • in each step, tree becomes one element of trees

Example 3: Storing results from a loop

Loops often create results, not just print things.

numbers <- 1:5
squares <- rep(NA, 6)

squares

for (i in numbers) {
  squares[i] <- i^2
}

squares

Explanation:

  • `rep(NA, 6)`` creates a vector of NA’s
  • squares[i] <- ... fills it step by step
  • indexing is essential when storing loop results
Group exercise: sum the squares vector (10 min)

tip use function sum, note that squares has 6 elements but only the first 5 are filled with values, thus you can either sum only the first 5 elements or use na.rm = TRUE inside the sum function to ignore NA values in the sum function.

To see the ‘sum’ funcion syntax, you can type ‘?sum’ in the console.

extra recreate the loop in example 3 but instead of creating a vector with 6 NA, use the function lenght to create a vector of the same length as the numbers vector, and fill it with NA values. This way you don’t have to manually specify the length of the squares vector.

If-statements: making decisions

An if-statement runs code only if a condition is TRUE.

Basic structure

if (condition) {
  code that runs if condition is TRUE
}

Conditions must evaluate to booleans i.e. TRUE or FALSE.

Example 4: A simple condition

x <- 5

if (x > 3) {
  print("x is larger than 3")
}

Because 5 > 3 is TRUE, the message is printed.

Using else

x <- 2

if (x > 3) {
  print("x is larger than 3")
} else {
  print("x is 3 or smaller")
}

Exactly one of the two blocks runs.

Multiple conditions with else if

x <- 5

if (x < 3) {
  print("small")
} else if (x < 10) {
  print("medium")
} else {
  print("large")
}

R checks conditions from top to bottom and stops at the first TRUE one.

Combining loops and if-statements

This is where things get useful.

Example 5: Classifying numbers

numbers <- c(2, 5, 8, 1)

for (n in numbers) {
  if (n < 5) {
    print(paste(n, "is small"))
  } else {
    print(paste(n, "is large"))
  }
}

Step-by-step:

  • loop over each number
  • check a condition
  • react differently depending on the result

Keeping it clean

In order to keep your workflow clean, it is recomended that intermediate results are saved to files. This way you can always go back to a previous step without re-running everything.

Also, save the scripts in separate steps, e.g. 01_load_data.R, 02_clean_data.R, 03_analyze_data.R, etc.

# save cleaned data to csv
write.csv(data_no_DeadWood, file = file.path(belizepath, "cleaned_tree_data.csv"), row.names = FALSE)
